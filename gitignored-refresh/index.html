readme update

you add all those files in .gitignore which we do not want git to track and sometimes you need to change the .gitignore file to not track/untrack some more files.

So if you change the gitignore file and do git add then the changes will not effected and the files will be tracked/untracked.

so there is the problem when we change(add or remove file) the gitignore later

suppose you want to add a new file in gitignore which you not want to track, now when you do git add the file will be tracked beacause of the git cache, which needs to be cleared when any changes occours in the gitignore file.

to git chache clear Run $ git rm -r –chached .
