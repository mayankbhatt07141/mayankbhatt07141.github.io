Docker

To install docker- https://docs.docker.com/engine/install/

Images

Images are blueprints for containers images are read only images contains

  1. Runtime environtment
  2. application code
  3. Any dependencies
  4. Extra configuration(e.g. env var.)
  5. etc.

layers of images

images are made up from several layers

  1. parent images: includes the os add sometimes the runtime environment(ex node)
  2. source code
  3. dependencies: A third party code/Software which our application depends upon
  4. run commands: these are the commands to run our application

cmd to create image

docker build --tag <name of image> .

command to delete image

docker rmi <image name or ID>

list all the images

docker images

Containers

containers are isolated Running instances of an image it runs our application

Run a container

docker run --<name for container> <image name or Id>

docker run --<name for container> <image name or Id> -p 4000:4000

here p is port and mapping port of host computer to docker container port

list all running containers

docker ps

stop a container

docker stop <container Id>

Dockerfile

Set of instruction to a docker to create the image and the docker file lists out all of these different layers or instructions to create those layers on image

name of file– Dockerfile

Docker compose

Compose is a tool for defining and running multi-container Docker applications
compose uses YAML file to configure our application service
name of file - docker-compose.yml

compose process

  1. define apps dockerfile so it will be reproduce from everywhere
  2. Define compose file docker-compose.yml so that all the containers can be run in isolated environment
  3. do docker-compose up this aggrigrates the output form each container

stop the container

docker-compose down
it stop the container and delete the cotainer but the image and the volumes are still reamaing

docker-compose down --rmi all -v
it removes images and volumes also.
